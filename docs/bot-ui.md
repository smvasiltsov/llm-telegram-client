# Управление ботом через Telegram

## Основной вход
Все настройки выполняются в личных сообщениях с ботом через команду `/groups`.

## Группы
- `groups` → выбор группы → список ролей.

## Роли
В карточке роли доступны:
- Skills;
- Pre/Post Processing;
- системный промпт;
- инструкция к сообщениям;
- инструкция для реплаев;
- LLM‑модель;
- переименование роли;
- сброс сессии;
- удаление роли.

## Skills
- В разделе `Skills` можно отдельно включать и выключать model-callable skills для конкретной роли.
- Сейчас доступны первые файловые skills:
  - `fs.read_file`
  - `fs.list_dir`
  - `fs.write_file`
- Для `fs.*` skills нужен `root_dir`:
  - если для роли уже задан provider field `working_dir`, он используется как `root_dir`;
  - иначе бот запросит путь в личных сообщениях как отдельный role-scoped field для skills.

## Pre/Post Processing
- `Pre/Post Processing` остается отдельным разделом.
- Это не skills для вызова моделью, а автоматические серверные pre/post hooks вокруг ответа LLM.

## Создание роли
При создании роли бот последовательно спрашивает:
1) имя роли;
2) описание;
3) системный промпт (можно пропустить);
4) выбор LLM‑модели.

## Как бот отвечает
- В группах бот реагирует только на сообщения владельца.
- Команды работают только в личке с ботом.
