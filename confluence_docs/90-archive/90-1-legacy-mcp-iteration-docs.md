# 90.1 Legacy MCP Iteration Docs

## Scope
This archive entry tracks legacy LTC MCP-oriented documents preserved for historical reference.

## Archived Source Set
Primary legacy MCP files:
- `docs/mcp/iter1-plan.md`
- `docs/mcp/skills-dev-guide.md`
- `docs/mcp/skills-sdk-v1.md`

## Historical Value
These documents capture earlier framing of:
- initial MCP and skills platform direction,
- early runtime assumptions,
- transitional developer guidance.

They remain useful for understanding design intent and terminology evolution.

## Current Position in Documentation Model
- Archive content is retained for traceability.
- It is not considered authoritative for current runtime behavior.
- Active specifications are documented in the main 00-80 sections.

## Recommended Usage
Use legacy MCP docs for:
- context during architecture reviews,
- interpreting old references in commits/issues,
- comparing prior and current contract assumptions.

Do not use them as the primary source for implementation decisions.

## Governance Rule
When legacy references become fully obsolete, they should stay in archive with clear pointers to replacement sections rather than being silently removed.
